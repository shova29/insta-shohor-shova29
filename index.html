JavaScript is known as a client-side scripting language. The term client-side scripting language which implies it runs at the client-side within the web-browsers, so the client's browser additionally has to support JavaScript. Inside every Browser, there is a JavaScript engine as V8 for Google Chrome and an environment to run JavaScript appropriately. Whenever we have a tendency to run a JavaScript code within an internet browser, JavaScript code is received by the browser's engine and also the engine runs the source code to get the output. The source code goes through four steps as (i) Parser, (ii) Abstract Syntax Tree, (iii) Conversion to Machine code, and(iv) Machine code and thus it gets executed in a standard JavaScript engine. JavaScript helps in making interactive websites. It supports functional, event-driven, prototype-based programming designs, and object-oriented. JavaScript significantly helps in DOM manipulation.

As we know that JavaScript includes a runtime model maintained an event loop, that responsible for gathering and processing events, executing the code and queued sub-tasks. The Event Loop has monitored the Call Stack and the Callback Queue. All the operations which will be executed in the code are kept tracked through the call stack. For completing the execution new functions to the stack are sent through the callback queue. When the call stack function is cleared, the Event Loop will bring out the first event from the queue and will push it into the Call Stack; which will efficiently run it in Last In First Out (LIFO). Such a cycle is called the event loop and this is how JavaScript handles its events. Each event is called a callback function. Through event loop, JavaScript provides us a misconception of being multithreaded however it is the single-threaded synchronous programming language.
